一、降级开启SSH方法 ​

BE3600买来就是最新的1.0.87版本（默认串口关闭），无法通过192.168.31.1后台降级固件，也无法通过网上的教程开启ssh并降级固件（下面有附带）。经过研究，终于发现可以通过下面方法搞定：

先说思路：在升级过程中断电，搞坏文件系统，路由器无法启动，再通过官方修复工具降级。

按照这个方法，我已成功将BE3600 2.5G v1.0.87版本降级到v1.0.81。

具体做法：

因为小米官方修复工具不能正常降级，会比较版本。所以只能用官方修复工具升级当前版本，也就是v1.0.87版本。 官方修复工具升级（小米路由器是双分区相互备份）：先升级root.ubi（第一分区系统），再升级backup of root.ubi（第二分区系统）

下面操作时机很重要 经过接了TTL线得出结论：在修复工具固件传输进度条结束后开始计时，12s左右断电（为了破坏第二分区系统），断电后需马上再次进入官方恢复流程，再次用修复工具升级， 第二次修复工具固件传输进度条结束后开始计时，5s左右断电（为了破坏第一分区系统）。

一定要按照先破坏OS2再破坏OS1顺序，这时OS1 ，OS2均被破坏。可以通过观察灯来确认路由器在不停重启。

接下来就可以通过官方修复工具进行降级了，三次修复工具升级，这次就用1.0.81固件，如果成功蓝灯闪烁，可以断电重启了，恭喜降级成功！如果橙色灯常亮，说明俩分区系统破坏不成功，肯定是断电时机不对。多试几次吧）。

然后通过小米SSH开启工具成功打开ssh，跟着视频操作！

二、刷入uboot和大分区表 ​

使用到的工具：SSH连接工具

这里我就以MobaXterm工具为例，打开SSH工具，填入路由器管理后台地址，登录用户为root，点击 OK 确定即可。

Uboot分区默认在mtd14里面，我们直接刷入就行了

写入Uboot

将下载的uboot和大分区文件直接拖拽传输至tmp文件夹。

cd /tmp

先检查文件是否上传成功 ls -l /tmp/*.bin

验证MD5（可选） md5sum /tmp/xiaomi-be3600-mibib.bin md5sum /tmp/xiaomi-be3600-u-boot.mbn

#刷大分区

mtd write xiaomi-be3600-mibib.bin /dev/mtd2

#刷uboot

mtd write xiaomi-be3600-u-boot.mbn /dev/mtd14

刷入完成然后重启：

reboot

如果想恢复小米原厂系统可以使用下面命令恢复！

cd /tmp

mtd write mtd2_0MIBIB.bin /dev/mtd2

mtd write mtd14_0APPSBL.bin /dev/mtd14

然后就是重启再用小米救砖工具刷就好了。

三、刷入OPENWRT固件 ​

设置电脑IP为 192.168.1.2 掩码为 255.255.255.0

路由器进入Uboot用卡针顶住路由器reset键，之后路由器接入电源。reset键一直按住等待10秒左右，松开reset键。

浏览器登录192.168.1.1 ，即可进入Uboot页面，上传openwrt固件即可。

OpenWrt
